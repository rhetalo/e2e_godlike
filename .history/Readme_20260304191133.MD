# e2e# Автотесты Playwright - Godlike Host

## Описание проекта

Этот проект содержит E2E (end-to-end) автотесты для веб-сайта [godlike.host](https://godlike.host) - хостинг-провайдера для игровых серверов (Minecraft, CS2, Rust и других).

## Основные понятия для начинающего QA

### Что такое E2E тесты?

E2E (End-to-End) тесты - это автоматические тесты, которые имитируют действия реального пользователя в браузере. Они проверяют, что весь функционал работает правильно от начала до конца.

### Что такое Playwright?

Playwright - это фреймворк (инструмент) для автоматизации браузера. Он позволяет:

- Писать тесты на JavaScript/TypeScript
- Запускать тесты в реальных браузерах (Chrome, Firefox, Safari)
- Делать скриншоты и записывать видео
- Автоматически ждать загрузки элементов
- Отлаживать тесты с помощью Trace Viewer

## Как запустить тесты

### Основные команды

```
bash
# Установить зависимости (выполняется один раз при первом запуске)
npm install

# Запустить все тесты (основная команда)
npx playwright test

# Запустить тесты и открыть HTML-отчет в браузере
npx playwright show-report

# Запустить тесты в интерактивном UI-режиме (удобно для отладки)
npx playwright test --ui

# Запустить конкретный тест по имени
npx playwright test "test base funnel"

# Запустить тесты только в одном браузере (chrome, firefox, webkit)
npx playwright test --project=chromium
```

## Основные команды Playwright (шпаргалка)

### Поиск элементов на странице

```
typescript
// Поиск по тексту
page.getByText('Текст')

// Поиск по роли элемента (рекомендуемый способ)
page.getByRole('button', { name: 'Название кнопки' })
page.getByRole('link', { name: 'Ссылка' })
page.getByRole('heading', { name: 'Заголовок' })
page.getByRole('textbox', { name: 'Поле ввода' })

// Поиск по CSS селектору
page.locator('.класс-элемента')
page.locator('#id-элемента')

// Поиск по placeholder
page.getByPlaceholder('Введите email')
```

### Действия с элементами

```
typescript
// Клик по элементу
await element.click()

// Ввод текста в поле
await element.fill('текст')

// Ожидание видимости элемента
await expect(element).toBeVisible()

// Проверка URL
await expect(page).toHaveURL('адрес')
await expect(page).toHaveURL(/часть-адреса/i)

// Ожидание появления элемента с таймаутом
await expect(element).toBeVisible({ timeout: 10000 })
```

## Описание тестов

### Тест 1: test base funnel

Проверяет полный процесс покупки Minecraft хостинга:

1. **Открытие сайта** - переход на godlike.host
2. **Выбор раздела** - клик по "Minecraft Server Hosting"
3. **Добавление в корзину** - выбор тарифа и нажатие "Add to Cart"
4. **Авторизация** - вход в существующий аккаунт
5. **Выбор локации** - выбор дата-центра для сервера
6. **Проверка заказа** - подтверждение на странице Review & Checkout

### Тест 2: check old funnel redirect

Проверяет, что старые ссылки (из писем, закладок) корректно перенаправляют на главную страницу.

## Конфигурация (playwright.config.ts)

Основные настройки:

- `testDir: './tests'` - где искать тестовые файлы
- `reporter: 'html'` - формат отчета (HTML)
- `trace: 'on-first-retry'` - записывать трассировку при падении теста
- `projects` - список браузеров для тестирования (Chromium, Firefox, WebKit)

## Зависимости (package.json)

Основные библиотеки:

- `@playwright/test` - основная библиотека Playwright
- `@types/node` - TypeScript типы для Node.js

## Советы для начинающих

1. **Начинайте с простого** - сначала запустите существующие тесты, посмотрите как они работают
2. **Используйте --ui режим** - для отладки очень помогает `npx playwright test --ui`
3. **Смотрите трассировку** - при падении теста смотрите Trace Viewer, там видно все действия
4. **Проверяйте селекторы** - используйте getByRole для надежных тестов
5. **Добавляйте wait'ы** - если элемент не находится, добавьте timeout

## Устранение проблем

### Тесты падают

1. Запустите с флагом `--ui` для визуальной отладки
2. Проверьте, что сайт доступен
3. Проверьте, что тестовые данные (логин/пароль) актуальны

### Нужно добавить новый тест

Создайте новый файл в папке `tests/` с расширением `.spec.ts`:

```typescript
import { test, expect } from "@playwright/test";

test("название теста", async ({ page }) => {
  // ваш код теста
});
```
